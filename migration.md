# Role and Objective
Act as an Expert Frontend Developer specializing in React and Next.js. Your task is to migrate an existing vanilla web project (built with plain HTML, CSS, and Vanilla JavaScript) into a modern Next.js application. 

# Core Requirements
1. **Framework & Architecture:** Use Next.js with the **App Router** (`app/` directory). Follow the latest official Next.js documentation and best practices for architecture and routing.
2. **Project Initialization:** Use the official Next.js initialization command (e.g., `npx create-next-app@latest .`) to scaffold the project directly in the **current directory**. 
3. **Fidelity:** Strictly preserve the original design, styles, animations, and custom business logic. The end result must look and behave exactly like the original, but powered by React/Next.js.
4. **Componentization:** Break down the monolithic HTML into reusable, logical React components.
5. **State & Logic:** Convert vanilla JavaScript DOM manipulations and event listeners into declarative React state (`useState`), side effects (`useEffect`), and event handlers (`onClick`, `onChange`, etc.).
6. **Optimization:** Utilize built-in Next.js optimizations where applicable (e.g., `next/image` for images, `next/link` for routing, `next/font` for typography).

# Specific Adaptations
* **Styling:** Maintain the original CSS structure. If the original uses global CSS, adapt it to Global Styles or CSS Modules (`*.module.css`) to prevent style leakage, whichever makes the most sense for the component architecture.
* **Animations:** Preserve all CSS transitions and keyframes. If there are JavaScript-based animations, adapt them using React `refs` (`useRef`) or suggest a lightweight React-friendly implementation if the vanilla approach conflicts with React's virtual DOM.
* **Client vs. Server Components:** By default, use React Server Components (RSC) to optimize performance. Only add the `"use client"` directive at the top of files that strictly require browser APIs, interactivity, or React hooks (like `useState` or `useEffect`).
* **Browser APIs:** Ensure any calls to `window`, `document`, or `localStorage` are safely wrapped inside `useEffect` or executed only on the client side to prevent hydration mismatches during Server-Side Rendering (SSR).

# Execution Steps
Please execute this migration in the following logical order:

### Phase 1: Assessment and Setup
1. Analyze the provided HTML, CSS, and JS files to understand the DOM structure, visual styling, and interactivity.
2. Run the official command to initialize the Next.js project in the current directory (`npx create-next-app@latest .`). If the CLI prompts about the directory not being empty, safely bypass it or temporarily move the source files to a subfolder to allow the installation, then move them back.
3. Configure the global layout (`app/layout.tsx` or `.jsx`) and clean up the boilerplate code generated by `create-next-app`.

### Phase 2: Structural Migration
1. Recreate the HTML skeleton by splitting it into modular React components (e.g., `Header`, `Footer`, `Hero`, `Sidebar`).
2. Implement the routing system based on the original HTML pages.

### Phase 3: Styling and Assets
1. Migrate the CSS. Ensure class names in JSX use the `className` attribute.
2. Move static assets (images, icons) to the `public/` directory and update their paths in the code using Next.js `<Image />` where appropriate for optimization.

### Phase 4: Logic and Interactivity
1. Translate vanilla JS functions into React components.
2. Replace `document.querySelector` and imperative DOM changes with React state variables and conditional rendering.
3. Handle component lifecycle and external API calls (if any) properly using Next.js data fetching patterns or React hooks.
